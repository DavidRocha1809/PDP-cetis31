2) Datos que se recopilan

Dependiendo del uso de la App, se puede recopilar lo siguiente:

2.1 Datos proporcionados por las personas usuarias

  • Correo institucional (requisito): el usuario proporciona su correo institucional para que se le dé de alta.
  • Datos de cuenta (según configuración): nombre, rol/perfil (por ejemplo: docente/administrativo) y datos asociados a su uso.
  • Soporte: información que envíes al contactar soporte (mensaje, descripción del problema, etc.).

2.2 Datos escolares gestionados dentro de la App

  • Información académica/administrativa (según funciones): grupos, listas, registros de asistencia, observaciones y/o reportes.
  • Datos de estudiantes (si se capturan/administran): nombre, matrícula/ID interno, grupo y asistencias (u otros campos que la institución utilice en la operación).

2.3 Archivos y documentos

  • Archivos que se suban a la App (por ejemplo: evidencias o documentos) cuando la función esté habilitada.

2.4 Datos técnicos mínimos

  • Información técnica básica para operar el servicio (por ejemplo: fechas/horas de acceso, identificadores técnicos necesarios para autenticación).

La App no solicita ubicación precisa, micrófono u otros permisos sensibles, salvo que en el futuro se habiliten funciones que lo requieran y se solicite permiso de forma explícita.

3) Finalidades del uso de la información

  • Dar de alta usuarios con correo institucional y administrar accesos.
  • Operar funciones principales: registro/consulta de asistencias, gestión de grupos/listas y generación de reportes.
  • Mantener seguridad del sistema (prevención de uso indebido y control de acceso por roles).
  • Atender solicitudes de soporte y comunicación operativa relacionada con el servicio.
  • Mejorar estabilidad y rendimiento de la App mediante revisión de incidencias (cuando aplique).

4) Uso de Firebase (Auth / Firestore / Storage)

La App utiliza servicios de Firebase para operar funcionalidades esenciales:

  • Firebase Authentication (Auth): para autenticación y administración de acceso.
  • Cloud Firestore: para almacenar información necesaria de operación (por ejemplo: usuarios, grupos, asistencias, reportes).
  • Cloud Storage for Firebase: para almacenar archivos y documentos subidos por usuarios (si aplica).

Importante: Firebase es provisto por Google y puede procesar datos en servidores ubicados fuera de México, dependiendo de la configuración del proyecto. Control Escolar C31 utiliza estos servicios para proveer el funcionamiento de la App.

Referencias de terceros (opcional para tu web):

5) Compartición de información

No vendemos tu información personal.

Podemos compartir información únicamente en estos supuestos:

  • Proveedores tecnológicos necesarios para operar la App (por ejemplo, Firebase), actuando como encargados del tratamiento.
  • Cumplimiento legal cuando una autoridad competente lo requiera conforme a ley.
  • Seguridad para investigar o prevenir fraude/abuso o incidentes técnicos.

6) Conservación de datos

  • Conservamos la información mientras sea necesaria para prestar el servicio y/o mientras la cuenta esté activa.
  • Cuando solicites eliminación, eliminaremos o anonimizaremos los datos, salvo que exista obligación legal o necesidad de conservación por seguridad o auditoría.
